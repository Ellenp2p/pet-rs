# pet-rs WASM 插件系统实现计划

## 项目概述

pet-rs 是一个基于 Bevy ECS 的虚拟宠物框架，支持 WASM 插件系统。

### 核心目标
1. ✅ 实现 WASM 插件运行时
2. ✅ 支持插件热重载
3. ✅ 实现插件间通信
4. ⏳ 实现插件生命周期管理
5. ⏳ 实现插件配置系统

## 当前架构

### 核心组件

```
src/
├── error.rs              # 错误类型
├── hooks/                # 钩子系统
├── network/              # 网络通道
├── plugins/              # 插件系统
│   ├── mod.rs
│   └── wasm_plugin.rs    # WASM 插件 Bevy 插件
├── wasm/                 # WASM 插件核心
│   ├── mod.rs
│   ├── bridge.rs         # 主机端插件管理
│   ├── plugin_trait.rs   # 插件 trait 定义
│   └── wasmtime_loader.rs # WASM 加载器
└── prelude.rs            # 公开 API
```

### 示例插件

```
examples/
├── basic_pet.rs          # 基础宠物示例
├── wasm_hooks/           # 示例 WASM 插件
├── wasm_stats/           # 统计插件
└── wasm_reader/          # 读取器插件
```

## 已完成功能

### 1. WASM 运行时
- ✅ wasmtime v24 支持
- ✅ WASM 模块加载和实例化
- ✅ 内存管理（64KB scratch buffer）
- ✅ 字符串传递协议

### 2. 热重载
- ✅ 同 ID 插件替换
- ✅ 不同 ID 插件共存
- ✅ 插件卸载

### 3. 插件状态存储
- ✅ 主机维护插件状态
- ✅ 插件状态持久化
- ✅ 状态读写接口

### 4. 插件间通信
- ✅ ABI 函数：`wasm_plugin_set_data`
- ✅ ABI 函数：`wasm_plugin_read_data`
- ✅ 数据存储：`HashMap<PluginId, HashMap<String, Vec<u8>>>`
- ✅ 回调函数机制

### 5. 统计插件
- ✅ 事件计数（购买、治疗、金币获得）
- ✅ 数据导出（通过 `wasm_plugin_set_data`）
- ✅ 内存布局（12字节：purchase_count + heal_count + gold_earned）

### 6. 读取器插件
- ✅ 数据读取（通过 `wasm_plugin_read_data`）
- ✅ 定时读取（每 10 帧）
- ✅ 事件触发读取
